# Analysis of Why the InferencePool Gateway Does Not Work

## Problem

The InferencePool Gateway returns 404 `route_not_found` and cannot route requests to ModelService pods.

## Root Cause

The InferencePool Gateway needs two key resources to work:

1. **InferencePool CRD instance** - defines which ModelService pods belong to the pool
2. **HTTPRoute resource** - configures how requests are routed to the InferencePool

In the current deployment:
- ✅ InferencePool Gateway pods are running
- ✅ Gateway API resources have been created
- ✅ ModelService pods have the correct labels (`llm-d.ai/inferenceServing=true`)
- ❌ **But there is no InferencePool CRD instance**
- ❌ **There is no HTTPRoute resource**

## How the InferencePool Gateway Works

```
Client Request
    ↓
InferencePool Gateway (Istio Envoy)
    ↓
HTTPRoute (routing rules)
    ↓
InferencePool CRD (defines which pods belong to the pool)
    ↓
ModelService Pods (matched by labels)
```

## Why Didn't Helmfile Create These Resources?

From the inspection results, helmfile deployed:
- `gaie-vllm-qwen2-5-0-5b` - InferencePool Extension (Endpoint Picker)
- `infra-vllm-qwen2-5-0-5b-inference-gateway-istio` - Gateway

However:
- The InferencePool CRD instance may need to be configured correctly in `gaie-inference-scheduling/values.yaml`
- The HTTPRoute may need to be created separately or generated by the helmfile template

## Solution

### Option 1: Use a Custom API Gateway (Recommended Right Now)

The Custom API Gateway already implements direct pod access and fallback behavior, so it does not depend on InferencePool Gateway auto-discovery.

### Option 2: Fix the InferencePool Gateway (Requires Extra Configuration)

You need to:
1. Create an InferencePool CRD instance that matches the ModelService pods
2. Create HTTPRoute resources that route requests to the InferencePool
3. Ensure the InferencePool Extension correctly discovers the ModelService pods

## Why Is This an Advantage of llm-d?

The InferencePool Gateway is a core llm-d advantage and provides:
- ✅ Intelligent routing and load balancing
- ✅ Prefix-cache awareness
- ✅ Automatic service discovery
- ✅ Production-grade reliability

However, it requires the InferencePool and HTTPRoute resources to be configured correctly before it will work.
